Activity Planning

A list of tasks is not a plan. A plan tells you which tasks depend on which other tasks, how long each will take, what the earliest possible finish date is, and which tasks — if they slip — will definitely make the whole project late. That's what this week is about.

01 · From tasks to network

The order of work matters as much as the work itself15 min

Imagine building a house. You can't lay the roof before the walls are up. You can't install plumbing before the rough framing is done — but you can paint the outside at the same time someone is installing the kitchen inside. The constraints of what comes before what, and what can happen in parallel, determine the minimum project duration more than the total hours of work do.

A network diagram makes these constraints visible. It's the foundation of all schedule planning.

FS
Finish-to-Start
Activity B cannot start until Activity A is finished. The most common dependency. "You can't test code that hasn't been written."
SS
Start-to-Start
Activity B can start as soon as Activity A starts. They can run in parallel once both are initiated. "Documentation can start as soon as coding starts."
FF
Finish-to-Finish
Activity B cannot finish until Activity A finishes. "Testing cannot finish until all bugs from development are fixed."

02 · Critical Path Analysis

Which tasks, if late, make the whole project late?25 min

The critical path is the longest sequence of dependent activities through the network. It defines the minimum possible project duration. Any delay on a critical path activity delays the entire project by the same amount. Tasks not on the critical path have "float" — a buffer of time they can slip without affecting the end date.

Finding the critical path involves two passes through the network:

Forward pass — find the Earliest dates
Start at the beginning (time 0) and work forward. For each activity, the Earliest Start (ES) is the latest finishing time of all its predecessors. Earliest Finish (EF) = ES + Duration. The EF of the final activity is the earliest possible project completion date.
Backward pass — find the Latest dates
Start at the project end date and work backward. The Latest Finish (LF) of each activity is the earliest Latest Start of all its successors. Latest Start (LS) = LF − Duration. Any activity with LS = ES (and LF = EF) is on the critical path.
Float (Slack)
Float = LS − ES = LF − EF. Float is the amount of time an activity can be delayed without delaying the project. Critical path activities have zero float — that's what makes them critical. Non-critical activities have positive float — a scheduling buffer.

PERT — adding uncertainty: Program Evaluation and Review Technique uses three time estimates per activity: Optimistic (O), Most Likely (M), Pessimistic (P). Expected duration = (O + 4M + P) / 6. This gives a weighted average that explicitly accounts for uncertainty — and can be used to calculate the probability of meeting a given deadline.

03 · Gantt Charts

Turning the network into a readable schedule15 min

A Gantt chart displays activities as horizontal bars on a time axis. Unlike a network diagram, it makes the calendar visible — actual dates, dependencies shown as bars starting where predecessors end, and resources often annotated along each bar. It's the most widely used scheduling tool in industry, and what most project management software (MS Project, Jira, Trello timelines) produces.

Gantt charts show time, not logic. You can see when things happen, but not why one task must follow another. In large projects, a Gantt chart without the underlying network is easy to misread. When someone proposes moving a bar to the left, always check the network first — it may be impossible due to dependencies.

04 · Wrap-up

Key takeaways

The critical path is where your attention belongs
Recovering from a delay on a non-critical activity costs float — which you may have plenty of. Recovering from a delay on a critical activity requires compressing the schedule by adding resources, removing scope, or accepting a later delivery date. The PM's job is to know which is which at all times.
Float is project oxygen
Float gives you flexibility to respond to the unexpected. Consuming all float early (by not enforcing deadlines on non-critical tasks) removes the buffer when you need it most, often turning formerly safe activities into critical ones.
Networks are inputs, Gantts are outputs
Build the logic in a network first. Use the critical path analysis to establish dates. Then produce the Gantt chart for communication. If you go straight to the Gantt, you're drawing a picture without understanding the dependencies beneath it.
